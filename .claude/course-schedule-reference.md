## Sprint Details

### Sprint 1: Ship & Showcase (Weeks 1-3)

**Goal:** Get comfortable shipping + build portfolio foundation

**Deliverables:**
- Portfolio site deployed (using Quarto template)
- About section complete
- 1 throwaway prototype deployed + added to portfolio
- "Problems I'm Exploring" page on portfolio:
  - 5 problems you personally experience
  - Framed professionally (public-facing)
  - Brief description of each + why it matters to you

**Portfolio sections after Sprint 1:**
- About Me
- Projects (1 prototype)
- Problems I'm Exploring

**Turn in:**
- 2-min Loom video (required)
- Portfolio URL (used all semester)

---

### Sprint 2: Validate & Commit (Weeks 3-5)

**Goal:** Validate a problem worth solving + start building real product

**Deliverables:**
- 5 problem interviews completed
- Interview synthesis on portfolio (use template):
  - Problem statement
  - Who you talked to (roles, not names)
  - Key insights (3-5 bullets)
  - Notable quotes
  - What you learned
- Problem statement locked
- Target customer defined
- First real prototype (v0.1) deployed
- Basic Supabase auth working

**Portfolio additions:**
- Featured Project section started
- Interview synthesis
- Problem statement
- Target customer
- v0.1 link

**Turn in:**
- 2-min Loom video (required)
- Portfolio updated

---

### Sprint 3: Core Product (Weeks 5-7)

**Goal:** Build the core feature that delivers your value proposition

**Deliverables:**
- PRD document (AI-assisted)
- Core feature functional with data persistence
- 3 user tests completed
- User feedback synthesis
- Platform strategy decision (web-only, PWA, or Capacitor)

**Portfolio additions:**
- PRD artifact (PDF or page)
- User feedback summary (polished)
- Updated product link
- Screenshots of core feature

**Turn in:**
- 2-min Loom video (required)
- Portfolio updated

---

### Sprint 4: Measure & Position (Weeks 7-9)

**Goal:** Know your numbers + prepare for launch

**Deliverables:**
- Analytics integrated + tracking key events
- Metrics dashboard (screenshot or link)
- Landing page live
- Go-to-market plan
- Product iteration based on data

**Portfolio additions:**
- GTM Plan artifact
- Metrics/results section started
- Landing page link
- Analytics dashboard screenshot

**Turn in:**
- 2-min Loom video (required)
- Portfolio updated

---

### Sprint 5: Monetize & Scale (Weeks 9-11)

**Goal:** Business model live + growth foundations

**Deliverables:**
- Business model canvas
- Pricing page/strategy implemented (or clear freemium model)
- Payment flow working (Stripe integration required)
- One growth experiment run
- Onboarding flow improved

**Portfolio additions:**
- Business Model artifact
- Results section updated
- Growth experiment results

**Turn in:**
- 2-min Loom video (required)
- Portfolio updated

---

### Final Project (Weeks 11-15)

**Goal:** Launch to real users + tell your story

**Deliverables:**
- Product live with real users (target: 5+, friends/family/classmates count)
- Final presentation deck
- 10-minute live demo + presentation (Week 14)
- Written reflection
- Graded on effort and learning, not user acquisition outcomes

**Portfolio additions:**
- Final results + metrics
- Technical details + GitHub
- Complete case study
- Everything polished

**Turn in:**
- 5-min Loom video (required)
- Presentation deck
- Portfolio complete

## Portfolio Structure

Students use the Quarto template from: https://github.com/byu-strategy/product-management-portfolio

**Structure:**
```
index.qmd           → About, skills, links
about.qmd           → Education, experience, goals
projects/
├── project-1.qmd   → Main product (case study)
├── project-2.qmd   → Throwaway prototype
└── project-3.qmd   → Throwaway prototype
```

**Main Project Case Study Sections:**
- The Problem
- The Solution
- Results
- Artifacts (PRD, GTM, Business Model)
- Technical Details
